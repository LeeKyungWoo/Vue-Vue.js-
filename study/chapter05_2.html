싱글 파일 컴포넌트 체계

.vue 파일 프로젝트 구조를 구성하는 방식을 말합니다.

확장자 .vue 파일 1개는 뷰 어플리케이션을 구성하는 1개의 컴포넌트와 동일합니다.

.vue 파일은 아래와 같은 기본 구조를 가집니다.

template 영역 : HTML 태그내용(화면에 표시할 요소들을 정의하는 영역, 예 : HTML+뷰 데이터 바인딩) script 영역 : 자바스크립트 내용(뷰 컴포넌트의 내용을 정의하는 영역, 예 : template, data, methods 등) style 영역 : css 스타일 내용(템플릿에 추가한 html 태그의 css 스타일을 정의하는 영역)
  <template>
        HTML 태그 내용
  </template>
  <script>
        자바 스크립트 내용
  </script>
  <style>
        CSS 스타일 내용
  </style>

뷰 CLI

싱글 컴포넌트 체계를 사용하기 위해서는 .vue 파일을 웹브라우저가 인식할 수 있는 형태로 변환줘야 합니다, 뷰는 뷰 코어팀에서 CLI 도구를 제공합니다. CLI는 커멘드 창에서 명령어로 특정 동작을 수행할 수 있는 도구인데, CLI에서 제공하는 명령어를 이용하면 뷰 어플리케이션을 개발하기 위한 초기 프로젝트 구졸르 쉽게 구성할 수 있습니다.

뷰 CLI 명렁어

  • vue init webapck : 고급 웹팩 기능을 활용한 프로젝트 구성 방식, 테스팅, 문법 검사 등을 지원
  • vue init webpack-simple : 웹팩 최소 기능을 활용한 프로젝트 구성 방식, 빠른 화면 프로토타이핑용
  • vue init browserify : 고급 브라우저리파이 기능을 활용한 프로젝트 구성 방식, 테스팅, 문법 검사 등을 지원
  • vue init browserify-simple : 브라우저리파이 최소 기능을 활용한 프로젝트 구성 방식, 빠른 화면 프로토타이핑용
  • vue init simple : 최소 뷰 기능만 들어간 HTML 파일 1개 생성
  • vue init pwa : 웹팩 기반의 프로그레시브 웹 앱 기능을 지원하는 뷰 프로젝트
